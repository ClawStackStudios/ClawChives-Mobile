Lifecycle Aware Collection: Replaced collectAsState with collectAsStateWithLifecycle throughout the Jetpack Compose UI (Dashboard and Gateway screens). Stateflows now logically pause and stop fetching when the app is docked or backgrounded—avoiding idle CPU drain from UI recompositions when the user is not actively interacting with the device screen.
Strict Rendering Laziness: Added unique identifiers (key = { it.id }) to the LazyColumn iterations in the Dashboard. When new Pinchmarks are inserted, deleted, or shifted, Compose will now efficiently skip redundant recompositions of the other Pinchmarks in the list.
On-Demand Demand Fetching: Verified that caching architectures rely purely on on-demand polling. State structures fetch directly from the network upon visual request cycles rather than constantly hammering background threads or polling continuously on an arbitrary timer scale.
In-Memory Sanitization: Interactions across filters, pots, tabs, and the search bar now instantly resolve against the pre-fetched local state machine. This eliminates continuous network telemetry loops per keystroke and preserves your exact scrolled position.
Batched Synchronization: Network queries the server for changes only during explicit reset cascades — such as an initial launch, pull-to-refresh, or immediately following an add or update mutation.
Lifecycle Efficiency & Render Opt-Outs: UI collections now strictly respect Android backgrounds via lifecycle-aware state observers (collectAsStateWithLifecycle), automatically pausing updates when out of focus. All primary visual lists (LazyColumn rendering) have also been updated to utilize stable object keys, actively dropping re-renders if the underlying item has not changed.
